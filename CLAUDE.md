## Project Overview

**StageConnect** is an internship-matching platform connecting students with companies. It is a Laravel 12 + React (Inertia.js) monolith serving as the web frontend; the long-term backend plan (see `docs/StageConnect_Backend_Plan.md`) targets a separate REST API built with Laravel + Sanctum + PostgreSQL + Redis + Meilisearch.

Three user roles: `student`, `company`, `admin`.

## Commands

### Development

```bash
composer dev          # Start all processes: PHP server, queue listener, Vite
```

### Building

```bash
npm run build         # Production build
npm run build:ssr     # SSR production build
```

### Testing

```bash
php artisan test                             # Run all tests (also runs pint lint check first)
php artisan test --filter TestName           # Run a single test
php artisan test tests/Feature/Auth/         # Run a specific folder
composer test                                # Same as above (via composer script)
```

Tests use Pest with `RefreshDatabase`. Feature tests extend `Tests\TestCase` (configured in `tests/Pest.php`). The test environment uses SQLite in-memory (`DB_DATABASE=:memory:`).

### Linting & Formatting

```bash
composer lint           # PHP: pint --parallel (auto-fix)
composer lint:check     # PHP: pint --parallel --test (CI mode)
npm run lint            # JS/TS: eslint --fix
npm run lint:check      # JS/TS: eslint (no fix)
npm run format          # Prettier write
npm run format:check    # Prettier check
npm run types:check     # tsc --noEmit
```

Pint uses the `laravel` preset (`pint.json`).

### Full CI check

```bash
composer ci:check     # lint:check + format:check + types:check + tests
```

## Architecture

### Backend (Laravel 12)

- **Auth**: Laravel Fortify (2FA, email verification, password reset). Auth routes are registered automatically by Fortify; UI lives in `resources/js/pages/auth/`.
- **Routing**: `routes/web.php` (main) + `routes/settings.php` (profile/password/appearance/2FA settings). Routes use Inertia's `Route::inertia()` helper where no controller logic is needed. Type-safe route helpers auto-generated by **Wayfinder** into `resources/js/wayfinder/`.
- **Controllers**: `app/Http/Controllers/Settings/` handles profile, password, and two-factor settings.
- **Concerns**: `app/Concerns/` holds reusable traits (e.g., `PasswordValidationRules`, `ProfileValidationRules`).
- **Actions**: `app/Actions/Fortify/` contains Fortify action implementations.

### Frontend (React + Inertia.js)

Stack: React 19, TypeScript, Tailwind CSS v4, shadcn/ui (Radix UI primitives), Lucide icons, Vite 7.

- **Pages** (`resources/js/pages/`): Inertia page components, resolved by name (e.g. `'settings/profile'` → `pages/settings/profile.tsx`).
- **Layouts** (`resources/js/layouts/`): `app-layout.tsx` (authenticated sidebar shell), `auth-layout.tsx` (unauthenticated centered card).
- **Components** (`resources/js/components/`): shared UI components; `ui/` sub-folder contains shadcn/ui primitives.
- **Hooks** (`resources/js/hooks/`): custom React hooks, including `use-appearance` for light/dark theme.
- **Route helpers** (`resources/js/wayfinder/`): auto-generated by Laravel Wayfinder on `npm run dev`/`build`. Do not edit manually.

### Planned Domain Model (see `docs/StageConnect_Backend_Plan.md`)

Key entities and relationships:
```
users ──── student_profiles ──── student_skill ──── skills
users ──── company_profiles ──── offers ──── offer_skill ──── skills
                                 offers ──── applications ──── messages
                                             applications ──── reviews
company_profiles ──── subscriptions (free / starter / pro)
```

Future stack additions: PostgreSQL, Redis, Meilisearch (Scout), Sanctum API tokens, Horizon, spatie/laravel-permission, spatie/laravel-medialibrary, Stripe/CinetPay.

## Key Conventions

- **PHP style**: PSR-12 enforced by Pint with the Laravel preset.
- **TypeScript**: strict mode; `@/` alias maps to `resources/js/`.
- **React Compiler** (`babel-plugin-react-compiler`) is enabled — avoid manual `useMemo`/`useCallback` unless the compiler can't handle the case.
- **Inertia shared data**: props passed from controllers are typed via `resources/js/types/`.
- **Wayfinder**: use generated route helpers from `@/wayfinder/` instead of hardcoded URL strings.
